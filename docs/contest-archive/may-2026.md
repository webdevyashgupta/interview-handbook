### Q1. Digit Frequency Score
**Difficulty:** 🟢 Easy | **Pattern:** Hash Map / Counting

**Intuition:**
Calculate the score based on the frequency of each digit in the input number. The score is the sum of (digit * frequency).

**Implementation:**
```python
def digitFrequencyScore(n):
    from collections import Counter
    counts = Counter(str(n))
    return sum(int(d) * freq for d, freq in counts.items())
```

### Q2. Maximum Number of Items From Sale I
**Difficulty:** 🟡 Medium | **Pattern:** Greedy / Sorting

**Intuition:**
Buying an item $I$ with factor $F_I$ gives free copies of all items $J$ where $F_I$ divides $F_J$. Group items by factor and pick the cheapest for each factor to maximize total items within budget.

**Implementation:**
```python
def maxItems(items, budget):
    min_prices = {}
    for factor, price in items:
        min_prices[factor] = min(min_prices.get(factor, float('inf')), price)
    
    sorted_factors = sorted(min_prices.items(), key=lambda x: x[1])
    count = 0
    bought_factors = set()
    for f, p in sorted_factors:
        if budget >= p:
            budget -= p
            bought_factors.add(f)
    
    total_items = 0
    unique_factors = set(f for f, p in items)
    for uf in unique_factors:
        if any(uf % bf == 0 for bf in bought_factors):
            total_items += 1
    return total_items
```

### Q3. Maximum Number of Items From Sale II
**Difficulty:** 🔴 Hard | **Pattern:** DP / Set Cover (Greedy approximation)

**Intuition:**
When budget allows buying multiple items and dependencies are complex, we use a greedy approach to pick the item that provides the maximum "newly covered" items per unit cost.

**Implementation:**
```python
def maxItemsII(items, budget):
    # items: list of (factor, price)
    factors = sorted(list(set(f for f, p in items)))
    costs = {f: float('inf') for f in factors}
    for f, p in items: costs[f] = min(costs[f], p)
    
    # dependencies[i] = list of factors that factor i covers
    deps = {f: [target for target in factors if target % f == 0] for f in factors}
    
    covered = set()
    total_bought = 0
    while budget > 0:
        best_f = -1
        best_ratio = 0
        for f, price in costs.items():
            if price <= budget:
                newly_covered = len([d for d in deps[f] if d not in covered])
                if newly_covered / price > best_ratio:
                    best_ratio = newly_covered / price
                    best_f = f
        if best_f == -1: break
        budget -= costs[best_f]
        for d in deps[best_f]: covered.add(d)
    return len(covered)
```

### Q1. Limit Occurrences
**Difficulty:** 🟢 Easy | **Pattern:** String / Counting

**Intuition:**
Remove characters from a string so that each character appears at most $k$ times. To keep the string as long as possible, process from left to right and keep track of counts.

**Implementation:**
```python
def limitOccurrences(s, k):
    counts = {}
    res = []
    for char in s:
        counts[char] = counts.get(char, 0) + 1
        if counts[char] <= k:
            res.append(char)
    return "".join(res)
```

### Q2. Password Strength
**Difficulty:** 🟡 Medium | **Pattern:** String / DP

**Intuition:**
A password's strength is the sum of "strengths" of all its substrings. Strength of a substring is the number of distinct characters. Use the contribution of each character approach.

**Implementation:**
```python
def passwordStrength(s):
    n = len(s)
    last_pos = {}
    total = 0
    curr = 0
    for i, char in enumerate(s):
        curr += (i - last_pos.get(char, -1))
        total += curr
        last_pos[char] = i
    return total
```

### Q3. Minimum Operations to Sort Permutation
**Difficulty:** 🟡 Medium | **Pattern:** Cyclic Sort / Greedy

**Intuition:**
The minimum number of swaps to sort a permutation is $N - \text{number of cycles}$. 

**Implementation:**
```python
def minSwaps(nums):
    n = len(nums)
    visited = [False] * n
    swaps = 0
    for i in range(n):
        if not visited[i]:
            curr = i
            cycle_size = 0
            while not visited[curr]:
                visited[curr] = True
                curr = nums[curr]
                cycle_size += 1
            swaps += (cycle_size - 1)
    return swaps
```

### Q4. Pairs After Increment
**Difficulty:** 🔴 Hard | **Pattern:** Math / Combinatorics

**Intuition:**
To maximize pairs $(a, b)$ such that $(a+b) \pmod T = 0$, we balance remainders $r$ and $T-r$. Each increment cost must be minimized.

**Implementation:**
```python
def maxPairs(nums, target, k):
    from collections import Counter
    counts = Counter(x % target for x in nums)
    # Use greedy or DP to decide which remainders to increment
    # given k total increments allowed.
    res = counts[0] // 2
    for r in range(1, (target // 2) + 1):
        if r == target - r:
            res += counts[r] // 2
        else:
            res += min(counts[r], counts[target - r])
    return res
```

### Q3. Maximum Path Intersection Grid
**Difficulty:** 🟡 Medium | **Pattern:** DP / Matrix

**Intuition:**
Maximize shared cells between two paths. $dp(k, r1, r2)$ where $k$ is steps, $r1, r2$ are rows of two paths.

**Implementation:**
```python
def maxIntersection(grid):
    n = len(grid)
    dp = [[-1] * n for _ in range(n)]
    dp[0][0] = grid[0][0]
    for k in range(1, 2*n - 1):
        new_dp = [[-1] * n for _ in range(n)]
        for r1 in range(max(0, k-n+1), min(k, n-1) + 1):
            for r2 in range(max(0, k-n+1), min(k, n-1) + 1):
                c1, c2 = k - r1, k - r2
                for pr1 in [r1, r1-1]:
                    for pr2 in [r2, r2-1]:
                        if pr1 >= 0 and pr2 >= 0 and dp[pr1][pr2] != -1:
                            val = grid[r1][c1] + (grid[r2][c2] if r1 != r2 else 0)
                            new_dp[r1][r2] = max(new_dp[r1][r2], dp[pr1][pr2] + val)
        dp = new_dp
    return dp[n-1][n-1]
```

### Q4. Count Non-Adjacent Subsets Tree
**Difficulty:** 🔴 Hard | **Pattern:** Tree DP

**Intuition:**
$dp[node][0]$ = ways to pick non-adjacent subset in subtree without picking $node$. $dp[node][1]$ = ways picking $node$.

**Implementation:**
```python
def countSubsets(root):
    def dfs(node):
        if not node: return 1, 0
        l_ex, l_in = dfs(node.left)
        r_ex, r_in = dfs(node.right)
        
        # Current node excluded: children can be either included or excluded
        ex = (l_ex + l_in) * (r_ex + r_in)
        # Current node included: children MUST be excluded
        inc = l_ex * r_ex
        return ex, inc
    
    res_ex, res_in = dfs(root)
    return res_ex + res_in
```

Find $x$ such that $x^k \le N$. Use binary search to avoid precision issues with `pow(n, 1/k)`.

**Implementation:**

```python
def countRoots(n, k):
    low, high = 1, n
    ans = 0
    while low <= high:
        mid = (low + high) // 2

            ans = mid
            low = mid + 1
        else:
            high = mid - 1
    return ans
```

```python
def largestLocal(grid):
```

### Q4. Smallest Unique Subarray
**Difficulty:** 🔴 Hard | **Pattern:** Monotonic Stack / Greedy

**Intuition:**
Use a monotonic stack to keep elements in increasing order, but only pop if the element will appear again later.

**Implementation:**
```python
def smallestUniqueSubarray(nums):
    last_occurrence = {x: i for i, x in enumerate(nums)}
    stack = []
    visited = set()
    for i, x in enumerate(nums):
        if x not in visited:
            while stack and x < stack[-1] and i < last_occurrence[stack[-1]]:
                visited.remove(stack.pop())
            stack.append(x)
            visited.add(x)
    return stack
```

### Q3. Minimize Array Sum Divisible
**Difficulty:** 🟡 Medium | **Pattern:** Greedy / Priority Queue

**Intuition:**
We can change elements to minimize the array sum such that it is divisible by $K$. Use a heap to store potential reductions.

**Implementation:**
```python
import heapq
def minSumDivisible(nums, k, ops):
    heap = [-(x - (x // k * k)) for x in nums]
    heapq.heapify(heap)
    total_sum = sum(nums)
    for _ in range(ops):
        red = -heapq.heappop(heap)
        total_sum -= red
        heapq.heappush(heap, 0)
    return total_sum
```

### Q4. Minimum Cost to Buy Apples II
**Difficulty:** 🔴 Hard | **Pattern:** Graph / Dijkstra

**Intuition:**
Multi-source Dijkstra where each node $i$ starts with distance $applePrices[i]$.

**Implementation:**
```python
import heapq
def minCostApples(n, roads, applePrices, k):
    adj = [[] for _ in range(n + 1)]
    for u, v, w in roads:
        adj[u].append((v, w * (k + 1)))
        adj[v].append((u, w * (k + 1)))
    
    pq = [(p, i) for i, p in enumerate(applePrices, 1)]
    heapq.heapify(pq)
    dist = {i: p for i, p in enumerate(applePrices, 1)}
    
    while pq:
        d, u = heapq.heappop(pq)
        if d > dist[u]: continue
        for v, w in adj[u]:
            if dist[v] > d + w:
                dist[v] = d + w
                heapq.heappush(pq, (dist[v], v))
    return [dist[i] for i in range(1, n + 1)]
```

### Q2. Minimum Flips to Coherent String

A coherent string has all '1's before all '0's. Count flips for each possible split point.

**Implementation:**

```python
def minFlips(s):
    n = len(s)
    ones_before = [0] * (n + 1)
    zeros_after = [0] * (n + 1)
    for i in range(n):
        ones_before[i+1] = ones_before[i] + (1 if s[i] == '1' else 0)
    for i in range(n - 1, -1, -1):
        zeros_after[i] = zeros_after[i+1] + (1 if s[i] == '0' else 0)
    return min(ones_before[i] + zeros_after[i] for i in range(n + 1))
```

```python
from collections import deque
def minGenerations(target):
    tx, ty, tz = target
    q = deque([(0, 0, 0, 0)])
    visited = {(0, 0, 0)}
    while q:
        x, y, z, d = q.popleft()
        if (x, y, z) == (tx, ty, tz): return d
```

### Q4. Minimum Threshold Path
**Difficulty:** 🔴 Hard | **Pattern:** Binary Search + BFS/Dijkstra

**Intuition:**
Minimize the maximum edge weight on a path. Binary search on the weight and use BFS for reachability.

**Implementation:**
```python
def minThreshold(n, edges, start, end):
    def can_reach(limit):
        adj = [[] for _ in range(n)]
        for u, v, w in edges:
            if w <= limit:
                adj[u].append(v)
                adj[v].append(u)
        q = [start]
        visited = {start}
        for u in q:
            if u == end: return True
            for v in adj[u]:
                if v not in visited:
                    visited.add(v); q.append(v)
        return False

    low = min(e[2] for e in edges)
    high = max(e[2] for e in edges)
    ans = high
    while low <= high:
        mid = (low + high) // 2
        if can_reach(mid):
            ans = mid; high = mid - 1
        else:
            low = mid + 1
    return ans
```

### Q4. Maximize Fixed Points
**Difficulty:** 🔴 Hard | **Pattern:** DP / LIS

**Intuition:**
Maximize $nums[i] = i$. Transform to $arr[i] = i - nums[i]$ and find LIS where $arr[i] \ge 0$.

**Implementation:**
```python
import bisect
def maximizeFixedPoints(nums):
    arr = [i - x for i, x in enumerate(nums) if i - x >= 0]
    # Standard LIS O(N log N)
    tails = []
    for x in arr:
        idx = bisect.bisect_right(tails, x)
        if idx < len(tails): tails[idx] = x
        else: tails.append(x)
    return len(tails)
```
