    99. Climbing Stairs¶

    LeetCode #70 | Difficulty: 🟢 Easy

    Problem Statement¶

    You are climbing a staircase with n steps. Each time you can climb 1 or 2 steps. Return the number of distinct ways you can climb to the top.

    Input:  n = 2
Output: 2  (1+1, 2)

Input:  n = 3
Output: 3  (1+1+1, 1+2, 2+1)

    Intuition¶

    To reach step n, you either came from step n-1 (1 step) or step n-2 (2 steps). So ways(n) = ways(n-1) + ways(n-2) — this is exactly the Fibonacci sequence.

    Base cases: ways(1) = 1, ways(2) = 2.

    Solution¶

    def climbStairs(n: int) -> int:
    if n <= 2:
        return n
    prev, curr = 1, 2
    for _ in range(3, n + 1):
        prev, curr = curr, prev + curr
    return curr

    Time: O(n) | Space: O(1)

    Interview Traps¶

    Trap 1 — Recursive without memoization Pure recursion is O(2^n) — identical to naive Fibonacci. Always add memoization or use bottom-up DP.

    Trap 2 — Off-by-one in base cases n=1 → 1 way, n=2 → 2 ways. Make sure your initialization matches.

    Trap 3 — "What if you can take up to k steps at a time?" Generalize: ways(n) = sum(ways(n-i) for i in range(1, k+1)). Use a sliding window sum over the dp array.

    Trap 4 — "What if each step has a cost?" (Min Cost Climbing Stairs) That's the next problem — know the distinction.

    100. Min Cost Climbing Stairs¶

    LeetCode #746 | Difficulty: 🟢 Easy

    Problem Statement¶

    Given an integer array cost where cost[i] is the cost to step from stair i, return the minimum cost to reach the top. You can start from index 0 or 1 and can climb 1 or 2 steps.

    Input:  cost = [10,15,20]
Output: 15

Input:  cost = [1,100,1,1,1,100,1,1,100,1]
Output: 6

    Intuition¶

    dp[i] = minimum cost to reach stair i. At each step: dp[i] = cost[i] + min(dp[i-1], dp[i-2]). The top is one step beyond the last stair — return min(dp[-1], dp[-2]).

    Solution¶

    def minCostClimbingStairs(cost: list[int]) -> int:
    n = len(cost)
    prev2, prev1 = cost[0], cost[1]

    for i in range(2, n):
        curr = cost[i] + min(prev1, prev2)
        prev2, prev1 = prev1, curr

    return min(prev1, prev2)

    Time: O(n) | Space: O(1)

    Interview Traps¶

    Trap 1 — Including cost of the "top" step The top is beyond the array — there's no cost to land there. Return min(dp[n-1], dp[n-2]).

    Trap 2 — Starting index You can start at index 0 OR index 1 (free). Both base cases must be initialized correctly.

    Trap 3 — Confusing with Climbing Stairs This problem has a cost at each step; Climbing Stairs just counts ways. Know the difference.

    101. House Robber¶

    LeetCode #198 | Difficulty: 🟡 Medium

    Problem Statement¶

    You are a robber. Given an array nums of house values, you cannot rob two adjacent houses. Return the maximum amount you can rob.

    Input:  nums = [1,2,3,1]
Output: 4  (rob house 1 and 3)

Input:  nums = [2,7,9,3,1]
Output: 12  (rob house 1, 3, 5)

    Intuition¶

    At each house, decide: rob it (and skip the previous) or skip it (and keep whatever was best up to the previous house).

    dp[i] = max(dp[i-1], dp[i-2] + nums[i])

    Optimize to O(1) space by keeping only the last two values.

    Solution¶

    def rob(nums: list[int]) -> int:
    prev2, prev1 = 0, 0

    for num in nums:
        curr = max(prev1, prev2 + num)
        prev2, prev1 = prev1, curr

    return prev1

    Time: O(n) | Space: O(1)

    Interview Traps¶

    Trap 1 — Greedy approach (taking every other house) Greedy doesn't work — the optimal solution may skip two consecutive houses. DP is required.

    Trap 2 — Off-by-one in initialization Initialize prev2 = 0, prev1 = 0 (zero houses robbed). The recurrence handles the rest.

    Trap 3 — "What if houses are in a circle?" (House Robber II) Run the DP twice: once on nums[:-1], once on nums[1:], and take the max. That's the next problem.

    Trap 4 — Single house nums = [5] → return 5. The loop runs once: curr = max(0, 0+5) = 5. Correct.

    102. House Robber II¶

    LeetCode #213 | Difficulty: 🟡 Medium

    Problem Statement¶

    Same as House Robber, but the houses form a circle — the first and last houses are adjacent.

    Input:  nums = [2,3,2]
Output: 3

Input:  nums = [1,2,3,1]
Output: 4

    Intuition¶

    Since first and last are adjacent, we can't rob both. Split into two cases: 1. Rob from index 0 to n-2 (exclude last). 2. Rob from index 1 to n-1 (exclude first).

    Return the max of both cases using the same House Robber I logic.

    Solution¶

    def rob(nums: list[int]) -> int:
    if len(nums) == 1:
        return nums[0]

    def rob_linear(houses):
        prev2, prev1 = 0, 0
        for num in houses:
            curr = max(prev1, prev2 + num)
            prev2, prev1 = prev1, curr
        return prev1

    return max(rob_linear(nums[:-1]), rob_linear(nums[1:]))

    Time: O(n) | Space: O(1)

    Interview Traps¶

    Trap 1 — Forgetting the single element case nums = [3] has no adjacency constraint — return nums[0] directly.

    Trap 2 — Running DP on full array You must exclude either the first or last element. Running on the full array double-counts the circular adjacency.

    Trap 3 — "Why two separate runs and not one?" You can't include both first and last. Splitting ensures only one is ever included in any optimal solution.

    103. Longest Palindromic Substring¶

    LeetCode #5 | Difficulty: 🟡 Medium

    Problem Statement¶

    Given a string s, return the longest palindromic substring.

    Input:  s = "babad"
Output: "bab"

Input:  s = "cbbd"
Output: "bb"

    Intuition¶

    Expand Around Center: for each character (and each pair of adjacent characters), expand outward while characters match. Track the longest palindrome found.

    This covers both odd-length (single center) and even-length (pair center) palindromes.

    Solution¶

    def longestPalindrome(s: str) -> str:
    start, end = 0, 0

    def expand(l, r):
        while l >= 0 and r < len(s) and s[l] == s[r]:
            l -= 1
            r += 1
        return l + 1, r - 1  # last valid l, r

    for i in range(len(s)):
        # Odd length
        l, r = expand(i, i)
        if r - l > end - start:
            start, end = l, r
        # Even length
        l, r = expand(i, i + 1)
        if r - l > end - start:
            start, end = l, r

    return s[start:end + 1]

    Time: O(n²) | Space: O(1)

    Interview Traps¶

    Trap 1 — Only checking odd-length palindromes Don't forget even-length palindromes (expand from between two chars).
    ✅ Always expand from both (i, i) and (i, i+1).

    Trap 2 — DP approach dp[i][j] = True if s[i..j] is a palindrome. Fill diagonally. O(n²) time and space — less elegant than expand-around-center.

    Trap 3 — Manacher's Algorithm follow-up Manacher's runs in O(n). Know it exists as a follow-up but the expand approach is expected in interviews.

    Trap 4 — Returning index vs substring The problem asks for the actual substring. Return s[start:end+1] (inclusive end).

    104. Palindromic Substrings¶

    LeetCode #647 | Difficulty: 🟡 Medium

    Problem Statement¶

    Given a string s, return the number of palindromic substrings.

    Input:  s = "abc"
Output: 3  ("a", "b", "c")

Input:  s = "aaa"
Output: 6  ("a","a","a","aa","aa","aaa")

    Intuition¶

    Same expand-around-center technique as Longest Palindromic Substring — but instead of tracking the longest, count every valid palindrome found during each expansion.

    Solution¶

    def countSubstrings(s: str) -> int:
    count = 0

    def expand(l, r):
        nonlocal count
        while l >= 0 and r < len(s) and s[l] == s[r]:
            count += 1
            l -= 1
            r += 1

    for i in range(len(s)):
        expand(i, i)      # odd length
        expand(i, i + 1)  # even length

    return count

    Time: O(n²) | Space: O(1)

    Interview Traps¶

    Trap 1 — Forgetting single-character palindromes Each single character is a palindrome. The expand(i, i) call counts it.

    Trap 2 — DP approach dp[i][j] = True if s[i..j] is a palindrome. Count all True entries. O(n²) space.

    Trap 3 — Confusing with Longest Palindromic Substring This problem counts ALL palindromes, not just the longest. The expand logic is the same but the aggregation differs.

    105. Decode Ways¶

    LeetCode #91 | Difficulty: 🟡 Medium

    Problem Statement¶

    A string of digits encodes letters where '1' → 'A', '2' → 'B', ..., '26' → 'Z'. Return the number of ways to decode the string.

    Input:  s = "12"
Output: 2  ("AB" or "L")

Input:  s = "226"
Output: 3  ("BZ","VF","BBF")

Input:  s = "06"
Output: 0  (leading zero)

    Intuition¶

    dp[i] = number of ways to decode s[0..i-1].

    At each position i: - If s[i-1] != '0' → single digit decode is valid: dp[i] += dp[i-1] - If s[i-2:i] is between "10" and "26" → two digit decode is valid: dp[i] += dp[i-2]

    Solution¶

    def numDecodings(s: str) -> int:
    if not s or s[0] == '0':
        return 0

    n = len(s)
    dp = [0] * (n + 1)
    dp[0] = 1  # empty string
    dp[1] = 1  # first character (guaranteed non-zero)

    for i in range(2, n + 1):
        # Single digit
        if s[i-1] != '0':
            dp[i] += dp[i-1]
        # Two digits
        two_digit = int(s[i-2:i])
        if 10 <= two_digit <= 26:
            dp[i] += dp[i-2]

    return dp[n]

    Time: O(n) | Space: O(n) → optimizable to O(1)

    Interview Traps¶

    Trap 1 — Not handling '0' correctly '0' cannot be decoded alone — it must pair with a preceding '1' or '2'.
    ✅ Check s[i-1] != '0' before adding dp[i-1].

    Trap 2 — Two-digit range Valid two-digit codes are 10–26. "27", "30", "00" are all invalid.

    Trap 3 — "30" case "30" — the '0' can't be decoded alone and 30 > 26 so no two-digit decode either → 0 ways.

    Trap 4 — O(1) space optimization Only dp[i-1] and dp[i-2] are needed. Use two variables instead of the full array.

    106. Coin Change¶

    LeetCode #322 | Difficulty: 🟡 Medium

    Problem Statement¶

    Given an array of coin denominations coins and an amount amount, return the fewest coins needed to make up amount. Return -1 if it's not possible.

    Input:  coins = [1,2,5], amount = 11
Output: 3  (5+5+1)

Input:  coins = [2], amount = 3
Output: -1

    Intuition¶

    Classic unbounded knapsack / BFS on amount. dp[i] = minimum coins to make amount i.

    For each amount, try every coin: dp[i] = min(dp[i], dp[i - coin] + 1) if i >= coin.

    Initialize dp[0] = 0, all others = infinity.

    Solution¶

    def coinChange(coins: list[int], amount: int) -> int:
    dp = [float('inf')] * (amount + 1)
    dp[0] = 0

    for i in range(1, amount + 1):
        for coin in coins:
            if coin <= i:
                dp[i] = min(dp[i], dp[i - coin] + 1)

    return dp[amount] if dp[amount] != float('inf') else -1

    Time: O(amount · len(coins)) | Space: O(amount)

    Interview Traps¶

    Trap 1 — Greedy approach (always pick largest coin) Greedy fails for many inputs, e.g., coins = [1, 3, 4], amount = 6 — greedy picks 4+1+1=3 coins but 3+3=2 is better.
    ✅ DP is required.

    Trap 2 — Initializing dp[0] incorrectly dp[0] = 0 (zero coins to make amount 0). This is the critical base case.

    Trap 3 — Returning 0 when no solution Return -1 when dp[amount] remains infinity, not 0.

    Trap 4 — "How many ways to make change?" (Coin Change II) That's LeetCode #518 — count combinations, not minimum coins. Requires a different recurrence.

    107. Maximum Product Subarray¶

    LeetCode #152 | Difficulty: 🟡 Medium

    Problem Statement¶

    Given an integer array nums, find the subarray with the largest product and return the product.

    Input:  nums = [2,3,-2,4]
Output: 6  ([2,3])

Input:  nums = [-2,0,-1]
Output: 0

    Intuition¶

    Unlike maximum sum subarray (Kadane's), negative numbers can flip max to min and vice versa. Track both max_prod and min_prod at each position — a negative number might turn the current minimum into the new maximum.

    Solution¶

    def maxProduct(nums: list[int]) -> int:
    result = max(nums)
    curr_min, curr_max = 1, 1

    for n in nums:
        if n == 0:
            curr_min, curr_max = 1, 1
            continue
        temp = curr_max * n
        curr_max = max(n * curr_max, n * curr_min, n)
        curr_min = min(temp, n * curr_min, n)
        result = max(result, curr_max)

    return result

    Time: O(n) | Space: O(1)

    Interview Traps¶

    Trap 1 — Only tracking max product A negative × negative = positive. Without tracking min_prod, you miss these cases.

    Trap 2 — Zeros reset the product 0 breaks any product chain. Reset both curr_min and curr_max to 1 when you encounter a zero.

    Trap 3 — Confusing with Maximum Subarray (Kadane's) Kadane's tracks running sum; this tracks running product. Negative numbers make products fundamentally different.

    Trap 4 — All negative numbers [-2, -3, -4] → max product is 12 (last two). Tracking both min and max handles this.

    108. Word Break¶

    LeetCode #139 | Difficulty: 🟡 Medium

    Problem Statement¶

    Given a string s and a dictionary wordDict, return true if s can be segmented into space-separated words all present in wordDict.

    Input:  s = "leetcode", wordDict = ["leet","code"]
Output: true

Input:  s = "applepenapple", wordDict = ["apple","pen"]
Output: true

    Intuition¶

    dp[i] = True if s[0..i-1] can be segmented using the dictionary. For each position i, check if any word in the dictionary ends at i and dp[i - len(word)] is True.

    Solution¶

    def wordBreak(s: str, wordDict: list[str]) -> bool:
    word_set = set(wordDict)
    n = len(s)
    dp = [False] * (n + 1)
    dp[0] = True  # empty string

    for i in range(1, n + 1):
        for word in word_set:
            w = len(word)
            if i >= w and dp[i - w] and s[i - w:i] == word:
                dp[i] = True
                break

    return dp[n]

    Time: O(n · m · k) where m = dict size, k = avg word length | Space: O(n)

    Interview Traps¶

    Trap 1 — Not converting wordDict to a set List lookup is O(n). Set lookup is O(1).

    Trap 2 — BFS/DFS alternative BFS from index 0 — each reachable position is a node. Add positions reachable by valid words. Equivalent to DP.

    Trap 3 — "Return all segmentations" (Word Break II) That's LeetCode #140 — backtracking with memoization. Much harder.

    Trap 4 — Duplicate words in dictionary Using a set de-duplicates automatically. No special handling needed.

    109. Longest Increasing Subsequence¶

    LeetCode #300 | Difficulty: 🟡 Medium

    Problem Statement¶

    Given an integer array nums, return the length of the longest strictly increasing subsequence.

    Input:  nums = [10,9,2,5,3,7,101,18]
Output: 4  ([2,3,7,101])

Input:  nums = [0,1,0,3,2,3]
Output: 4

    Intuition¶

    DP approach (O(n²)): dp[i] = length of LIS ending at index i. For each i, look back at all j < i where nums[j] < nums[i] and take max(dp[j]) + 1.

    Patience sorting (O(n log n)): Maintain a sorted list of "tails" of active subsequences. Binary search for the correct position to place each element.

    Solution¶

    DP — O(n²): 

    def lengthOfLIS(nums: list[int]) -> int:
    dp = [1] * len(nums)

    for i in range(1, len(nums)):
        for j in range(i):
            if nums[j] < nums[i]:
                dp[i] = max(dp[i], dp[j] + 1)

    return max(dp)

    Patience Sorting — O(n log n): 

    import bisect

def lengthOfLIS(nums: list[int]) -> int:
    tails = []

    for num in nums:
        pos = bisect.bisect_left(tails, num)
        if pos == len(tails):
            tails.append(num)
        else:
            tails[pos] = num

    return len(tails)

    Time: O(n log n) | Space: O(n)

    Interview Traps¶

    Trap 1 — Confusing subsequence with subarray A subsequence doesn't need to be contiguous — elements can be skipped.

    Trap 2 — Initializing dp with 0 instead of 1 Every element alone forms an LIS of length 1. Initialize dp = [1] * n.

    Trap 3 — Not understanding patience sorting The tails array doesn't store the actual LIS — it stores the smallest possible tail for each length. Its length equals the LIS length.

    Trap 4 — "What if we need the actual subsequence, not just length?" Reconstruct by tracking predecessors in the DP approach.

    110. Partition Equal Subset Sum¶

    LeetCode #416 | Difficulty: 🟡 Medium

    Problem Statement¶

    Given an integer array nums, return true if you can partition it into two subsets with equal sum.

    Input:  nums = [1,5,11,5]
Output: true  ([1,5,5] and [11])

Input:  nums = [1,2,3,5]
Output: false

    Intuition¶

    Equal sum partition iff total is even AND we can find a subset summing to total // 2. This reduces to a 0/1 knapsack problem: can we reach the target sum using a subset?

    dp[j] = True if subset sum j is achievable. For each number, update dp from right to left.

    Solution¶

    def canPartition(nums: list[int]) -> bool:
    total = sum(nums)
    if total % 2 != 0:
        return False

    target = total // 2
    dp = {0}  # achievable sums

    for num in nums:
        dp = dp | {s + num for s in dp}
        if target in dp:
            return True

    return target in dp

    Boolean array version: 

    def canPartition(nums: list[int]) -> bool:
    total = sum(nums)
    if total % 2:
        return False
    target = total // 2
    dp = [False] * (target + 1)
    dp[0] = True

    for num in nums:
        for j in range(target, num - 1, -1):  # traverse right to left!
            dp[j] = dp[j] or dp[j - num]

    return dp[target]

    Time: O(n · target) | Space: O(target)

    Interview Traps¶

    Trap 1 — Not checking if total is even If the total is odd, equal partition is impossible. Early return saves time.

    Trap 2 — Traversing left to right in 0/1 knapsack Left-to-right allows using the same element multiple times (unbounded knapsack).
    ✅ Always traverse right to left for 0/1 knapsack (each item used at most once).

    Trap 3 — "What if we can use each element multiple times?" That's unbounded knapsack — traverse left to right.

    Trap 4 — Large target values If target is very large (e.g., 10^5), the DP array is still manageable. Watch for problem constraints.
